# DOM Surgeon ✂️

![Version](https://img.shields.io/badge/version-1.0.0-blue)
![Platform](https://img.shields.io/badge/platform-Chrome%20Extension-success)
![License](https://img.shields.io/badge/license-MIT-green)

**DOM Surgeon** is a powerful, visual, and highly persistent Chrome Extension that allows you to surgically edit any website. You can visually select, hide, resize, and style any DOM element on the web. Unlike Chrome Developer Tools, DOM Surgeon saves your changes locally and gracefully syncs them across all your connected browsers in real-time.

## ✨ Core Features
* 🖱️ **Visual DOM Editing:** Hover over any element to highlight it. Click to instantly apply mutations (Hide, Resize, CSS styling).
* 🛡️ **Shadow Encapsulated UI:** The draggable floating widget and breadcrumb menus are built using modern Shadow DOM & HTML Popover APIs, ensuring host-website CSS never breaks the UI.
* 🌲 **Advanced Breadcrumb Traversal:** Click deeply nested wrapper elements precisely by traversing the DOM tree hierarchy in the top popover bar.
* 💾 **Persistent Local Storage:** Changes survive page reloads and browser restarts. The extension intelligently partitions rules by domain to ensure blazing fast performance.
* ☁️ **Tiered Cloud Synchronization:** 
  * *1-Minute Debounce:* Syncs securely back up your unlimited rule payloads directly to a GitHub Gist using a Personal Access Token without hitting rate limits.
* ⏳ **Cloud Time Machine:** Made a massive mistake? Use the native Git version-history built into GitHub Gists to roll back your entire extension state to any previous snapshot.

## 🚀 Installation

DOM Surgeon is designed for power users. To install it locally in Developer Mode:

1. Clone or download this repository to your local machine.
2. Open Google Chrome and navigate to `chrome://extensions/`.
3. Enable **Developer mode** using the toggle switch in the top right corner.
4. Click the **Load unpacked** button.
5. Select the `dom-surgeon` directory.
6. Pin the DOM Surgeon icon to your Chrome toolbar for easy access!

## 🛠️ Usage Guide

### On-Page Editing
1. Click the DOM Surgeon extension icon in your toolbar to activate the editor on any webpage.
2. A floating widget will appear. You can drag it anywhere on your screen.
3. Move your mouse around the webpage. A vibrant blue outline will highlight the exact DOM element you are targeting.
4. Click to select the element, then use the widget to:
   - **Hide:** Visually remove it (`display: none`).
   - **Resize:** Apply custom width/height values.
   - **Class Manipulation:** Inject custom CSS utility classes.
5. Made a mistake? Use `Ctrl+Z` to undo your local actions, or `Ctrl+Y` to redo.

### Dashboard Management
1. Right-click the DOM Surgeon extension icon and select **Dashboard**.
2. From the Dashboard, you can view a categorized list of every website you've modified.
3. Expand a website's accordion to delete individual rules, or click **Clear All Trash** to permanently execute garbage collection on soft-deleted elements.

## ☁️ Cloud Sync & Time Machine Setup

To unlock unlimited cross-browser sync and the Cloud Time Machine disaster recovery, you need to configure GitHub Gist integration.

### 1. Generate a GitHub PAT
1. Go to [GitHub Developer Settings](https://github.com/settings/tokens).
2. Click **Generate new token (classic)**.
3. Give it a name (e.g., "DOM Surgeon Sync"), set expiration to "No expiration", and check the **`gist`** scope box.
4. Generate and copy the token.

### 2. Link Your First Browser
1. Open the DOM Surgeon **Dashboard**.
2. Paste the token into the **Personal Access Token** field.
3. Leave the **Gist ID** field empty.
4. Click **Sync Now**. DOM Surgeon will automatically create a new hidden Gist on your GitHub account and populate the Gist ID field for you.

### 3. Link Additional Browsers
1. On your second computer/browser, install DOM Surgeon and open the Dashboard.
2. Paste the exact same **Personal Access Token**.
3. Copy and paste the **Gist ID** generated by your first browser into the Gist ID field.
4. Click **Sync Now**. Both browsers are now permanently locked into the same auto-syncing Cloud File!

**Note on Experimentation:** If you want to mess around without destroying your cloud data, simply click the **Pause Sync** button in the Dashboard.

### Permissions Justification
* `activeTab`: To inject the DOM manipulation content scripts into the active page when you click the extension icon.
* `storage`: To save your customized CSS selectors and rules locally.
* `contextMenus`: To provide the "Sync Cloud Now" and "Dashboard" quick actions.
* `alarms`: To run the highly efficient, batched 1-minute background timer for the GitHub Gist synchronization.
